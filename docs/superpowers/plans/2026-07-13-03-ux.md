# NEURAMED Phase 3 — Flagship UX Plan

> Subagent-driven. Goal: 5.5/10 → flagship, mobile-first, accessible — WITHOUT breaking the working data flow from Phases 1-2. Keep the cyan/green clinical-HUD brand identity; fix the execution.

**Constraint:** every task ends with `npx tsc --noEmit` clean + `npm run build` green. Frontend: `neural_care_hub_main/`.

## Audit-derived defects (the work)
- **Broken shadcn HSL token bridge** — `tailwind.config.ts` maps `background/primary/muted/border/ring/...` to `hsl(var(--…))` but `index.css` defines none of those tokens (and defines `--border: rgba()`, `--foreground: var(--hex)` — wrong type into `hsl()`). So `text-muted-foreground`, `bg-background`, `border-border`, `bg-primary` render **invalid → dropped by browser**. Two color systems fight (brand `#00E5FF` vs Tailwind `#06b6d4`).
- **Mobile broken by construction** — inline `style={{ gridTemplateColumns: '5fr 7fr' }}` / `repeat(4,1fr)` everywhere → Tailwind breakpoints can't reach → never collapse. Panels crush to ~150px on phones.
- **3 cursors, 1 dead, none touch-aware** → static cyan artifacts on phones; global `cursor:none !important` nukes all affordances.
- **No real `prefers-reduced-motion`**; LoginPage Three.js unguarded (up to 3 WebGL contexts on a phone, O(n²) per-frame rebuild).
- **A11y ≈ 0** — `--muted` 2.7:1 contrast (fails WCAG), div-buttons not keyboard-reachable, 8-11px text, <44px touch targets, unlabeled icon controls.
- **No type/spacing/radius scale**; dead `App.css`; 3 unused fonts render-blocking in `index.html`.

## Tasks
- **T1 — Color/token foundation.** Define the full shadcn token set in `index.css` as HSL triplets so `hsl(var(--x))` resolves; map brand cyan/green/amber/red into it as the single source of truth; fix `--muted`/`--dim` to ≥4.5:1 WCAG AA; add `--radius`, a type scale, a spacing rhythm. Reconcile the two color systems to one. Remove dead `App.css` + unused font `<link>`s (keep Geist).
- **T2 — Responsive layouts.** Replace inline `gridTemplateColumns`/`repeat(N,1fr)` across pages with responsive CSS (min/max auto-fill + `lg:` breakpoints); single-column forms under `sm`; tables → stacked cards on mobile; unify the 768px breakpoint (JS `use-mobile` + CSS agree). Verify no horizontal scroll on 375px.
- **T3 — Cursor + motion.** Delete the dead cursor; merge remaining two into one touch-aware component gated behind `matchMedia('(pointer:fine)')`; drop global `cursor:none`. Add a real `prefers-reduced-motion` strategy (Framer + CSS keyframes + count-ups + Three.js). Guard LoginPage Three.js behind the mobile check; fix the `useMiniScene` geometry/material leak.
- **T4 — Accessibility.** div-buttons/rows → real `<button>`/`role`+`tabIndex`+`onKeyDown`; `aria-label` on icon controls; `aria-current` on nav; 44px min touch targets; associate form labels; ensure focus-visible rings survive.
- **T5 — Provenance UI polish** (the signature surface) + final responsive/a11y verification pass.

Each task: read the real files first, keep data wiring intact, tsc+build green, commit. Detail lives in dispatch prompts.
